# เอกสารข้อกำหนดซอฟต์แวร์ (Software Requirement Specification - SRS)

## 1. บทนำ
- **ชื่อเกม**: Sara Adventure
- **ประเภท**: เกมผจญภัย 2 มิติ (2D Adventure) ใช้ Pygame
- **ผู้พัฒนา**: ทีมผู้เรียน OOP682
- **วัตถุประสงค์**: สร้างเกมที่ผู้เล่นควบคุมตัวละคร Sara ผ่านสองแผนที่เพื่อเก็บถ้วยรางวัลและจบเกม

## 2. คำอธิบายโดยรวม
- เกมประกอบด้วย **สองแผนที่** (`Map 1` และ `Map 2`) แต่ละแผนที่มีขนาด **20 x 20 tiles**
- **Map 1** ใช้ tileset ชื่อ **forest** (ภาพพื้นป่า)
- **Map 2** ใช้ tileset ชื่อ **space** (ภาพพื้นอวกาศ)
- ตัวละคร **Sara** จะเริ่มต้นที่ **Map 2** (ตำแหน่งเริ่มต้นกำหนดโดยผู้พัฒนา)
- ใน **Map 2** มี **ถ้วยรางวัล** (trophy) อยู่หนึ่งชิ้น เมื่อ Sara เก็บถ้วยได้ เกมจะแสดงข้อความแสดงความยินดีและจบเกม

## 3. ข้อกำหนดฟังก์ชัน (Functional Requirements)
| # | รายละเอียด | หมายเหตุ |
|---|------------|----------|
| 1 | แสดงแผนที่ 1 และ 2 ตาม tileset ที่กำหนด | ใช้ไฟล์ `forest_tileset.png` และ `space_tileset.png` |
| 2 | ตัวละคร Sara สามารถเคลื่อนที่ด้วยปุ่มลูกศร (↑, ↓, ←, →) | ความเร็วเคลื่อนที่กำหนดเป็นค่าคงที่ |
| 3 | ตรวจจับการชนของ Sara กับถ้วยรางวัลบน Map 2 | เมื่อชนให้แสดงข้อความ "ยินดีด้วย! คุณเก็บถ้วยรางวัลสำเร็จ" แล้วปิดเกม |
| 4 | ระบบเสียง (sound) – เล่นเสียงเมื่อกดปุ่ม Space (ถ้ามีไฟล์ `wave_sound.wav`) | หากไม่มีไฟล์เสียงให้แสดงข้อความข้อผิดพลาดแต่เกมยังทำงานต่อได้ |
| 5 | ระบบการวาด (render) – ใช้ `pygame.sprite.Group.draw` เพื่อวาด Sara และถ้วยรางวัลบนหน้าจอ |
| 6 | จัดการเหตุการณ์ปิดหน้าต่าง (QUIT) อย่างเหมาะสม | ปิดเกมโดยเรียก `pygame.quit()` และ `sys.exit()` |

## 4. ข้อกำหนดไม่ใช่ฟังก์ชัน (Non‑Functional Requirements)
- **ประสิทธิภาพ**: เกมควรทำงานที่อัตราเฟรมอย่างน้อย 30 FPS บนเครื่อง Windows 10/11
- **ความเข้ากันได้**: รองรับ Python 3.13+ และ Pygame 2.6.1
- **ความเสถียร**: ไม่มีการ crash จากการโหลดไฟล์ภาพหรือเสียงที่หายไป (ต้องจัดการข้อยกเว้น)
- **การออกแบบ UI**: หน้าต่างเกมมีชื่อ "Sara Adventure" และขนาด 800 x 600 พิกเซล

## 5. ข้อกำหนดข้อมูล (Data Requirements)
- **Tileset**: `forest_tileset.png`, `space_tileset.png` – แต่ละไฟล์ต้องเป็น PNG มีขนาดที่สอดคล้องกับการแบ่ง 20x20 tiles
- **Sprite Sheet**: `sara_spritesheet.png` – ใช้สำหรับแสดงภาพ Sara
- **Object**: `trophy.png` – ภาพถ้วยรางวัล
- **Sound** (optional): `wave_sound.wav`

## 6. การทดสอบ (Verification)
- **Test Case 1**: เปิดเกม → แสดง Map 2 (space) → Sara ปรากฏที่ตำแหน่งเริ่มต้น
- **Test Case 2**: เคลื่อนที่ Sara ไปยังตำแหน่งถ้วย → ตรวจสอบว่าข้อความแสดงความยินดีปรากฏและเกมจบ
- **Test Case 3**: ปิดหน้าต่างโดยคลิก X → ตรวจสอบว่าโปรแกรมปิดอย่างเรียบร้อย
- **Test Case 4**: กด Space → ตรวจสอบว่าเสียงเล่น (ถ้ามีไฟล์) หรือไม่มีการ crash หากไม่มีไฟล์

## 7. ข้อจำกัด (Assumptions & Constraints)
- ผู้เล่นจะใช้คีย์บอร์ดเป็นอินพุตหลัก
- ไฟล์ assets (tileset, sprite sheet, trophy, sound) จะอยู่ในโฟลเดอร์เดียวกับโค้ด (`./`)
- เกมจะทำงานบน Windows เท่านั้น (ตามสภาพแวดล้อมของผู้ใช้)
